How it works

AUGUR reads public ledger data, normalizes it, then outputs plain-English statements that map to verifiable facts. The architecture stays simple and safe so it can scale without custody risk.

1
Read public XRPL data

Wallet addresses are public. AUGUR reads what the ledger exposes and does not request keys.

2
Normalize and classify

Events get structured into consistent categories so the output stays predictable and useful.

3
Produce statements

The output is plain-English, easy to share, and backed by data you can verify.

4
Expose via API

Apps, dashboards, and operators consume the API. The site stays light and fast.

FAQ

Straight answers. No fluff.

Does AUGUR require my wallet to connect

No. Viewing and reading are based on public addresses. No wallet connect is needed for the read layer.

Does AUGUR ever handle keys

No. The design goal is read-only intelligence from public data, with zero custody.

What is the API endpoint

Use https://api.augurxrpl.com. Health checks are at /health and /extras/health.

Can this expand beyond XRPL later

Yes. XRPL is the focus. Expansion happens only after XRPL is locked and battle-tested.

Build with AUGUR

If you build tools, dashboards, or ecosystem utilities, the read layer is live and ready. Start with the health endpoints, then build your first statement workflow.
